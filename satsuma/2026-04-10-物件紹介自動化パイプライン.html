ステータス管理（G列）

スプレッドシート「物件紹介」タブのG列（ST）がパイプライン全体の進捗を管理します。A列フラグ=1の行に対し、G列の値に応じて適切なスキルを起動します。各工程完了後、A列のフラグは即座に0に書き換えられます。

運用メモ

⚠️ L列・M列は手動入力が必要

販売金額（L列）と路線価（M列）はシステムが自動取得できません。案件登録時に必ず入力してください。K-M列の欠損検知時は「要確認」として報告し、処理を停止せず次へ繋げます。

📁 ★共有フォルダで完結する運用

K列の物件フォルダ内の★共有フォルダで資料整理が完結します。墨消し作業は廃止され、担当者が事前に墨消し済みファイルを投入する運用に変更されました。

✅ A列フラグで差分実行

A列に「1」が入っている行のみ処理対象。完了後は自動で「0」に戻るため、二重実行を防止。途中で中断しても「販売マネージャー起動して」で続きから再開できます。

🧠 コンテキスト・カプセル化戦略

フェーズ別に情報を遮断し、実行中のフェーズに不要なスキル定義は読み込みません。巨大ファイル（base_coords_v6.json等）は使用時のみ参照し、処理後に即解放します。

トークン省エネ対策 4箇条
1
スキルの保存は1日1回に制限
スキルファイルの頻繁な保存はトークンを大量消費する。更新は1日1回の定期メンテナンスにとどめ、作業中の細かな修正は会話内メモで管理する。保存のたびにファイル全体の読み書きが発生するため、回数を減らすだけでトークンが大幅に節約できる。
2
スプレッドシートの無駄な行・列を削除
GASシートリーダーで読み込む範囲を最小化する。空白行・使わない列・過去の完了済み案件を定期的に削除またはアーカイブし、AIが読み込むデータ量そのものを減らす。A列=1の行に絞る差分実行との組み合わせで効果倍増。
3
スキルは作りすぎない・長くしない
スキルファイルは起動時に全文が読み込まれるため、数が多いほど・長いほどトークンを消費する。1スキルは必要最小限の記述に絞り、重複する手順は共通化する。未使用スキルは無効化またはアーカイブして常時ロードされないようにする。
4
作業前に「省エネできるか？」を確認
パイプライン実行前に「この作業で省エネできるポイントはないか？」をAI自身に問いかける運用を導入。不要なスキル読み込み・重複処理・過剰なChrome操作がないかを事前チェックし、実行計画を最適化してから着手する。

Risks & Mitigations
リスク影響対策
配配メール / Lステップ の UI 変更 高 Playwright セレクターを外部 YAML で管理。スクリーンショット差分監視で変更を早期検知。
Google OAuth トークン期限切れ 高 サービスアカウントを使用（有効期限なし）。オフラインリフレッシュトークンを定期更新。
Anthropic API コスト増大 中 HTML 生成プロンプト最適化。リトライ時の再生成防止キャッシュ。月次コスト上限アラート。

Security Design

🔑 認証情報の移行（最優先）

スキルファイルにハードコードされている GitHub PAT は即時 Secret Manager へ移行し、ファイルから削除してください。配配メール・Lステップのログイン情報も同様に集中管理します。

🛡 最小権限のサービスアカウント

Google サービスアカウントには roles/drive.file（指定フォルダのみ）・Sheets 読み書き・Secret Manager 特定シークレット読み取り の最小権限のみ付与します。

🌐 API エンドポイント保護

外部公開エンドポイントは Bearer Token (JWT) または Cloud IAM で保護。GAS Webhook は HMAC-SHA256 署名検証 を必須とします。

🍪 Playwright セッション管理

初回ログイン後のセッション Cookie を storage_state() で保存し Secret Manager に暗号化して格納。コード・ログへのパスワード出力は厳禁です。
