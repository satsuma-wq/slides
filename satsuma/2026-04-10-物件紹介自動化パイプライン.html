運用メモ

⚠️ K列・L列は手動入力が必要

販売金額（K列）と路線価（L列）はシステムが自動取得できません。案件登録時に必ず入力してください。未入力の場合、概要書作成開始前にシステムが確認を求めます。

📁 フォルダ構造は統一が前提

J列の物件フォルダの中には、必ず「共有フォルダ」と「バラバラフォルダ」の2つが存在する必要があります。概要書は共有フォルダ、資料整理作業はバラバラフォルダを使用します。

✅ 途中再開が可能

各ステップ完了時にF列（ST）が更新されるため、途中で処理が中断されても「販売マネージャー起動して」と指示するだけで続きから再開できます。

🔍 墨消しは目視確認が必要

資料整理の墨消しステップでは、処理内容を人の目で確認してから進めます。システムが自動で一時停止し確認を求めるため、見落としなく対応できます。

Technical Challenges

最大の課題は 配配メール・Lステップが公式 API を提供していない点。現行の Chrome UI 操作を Playwright ヘッドレスブラウザで代替します。

⚠️ Playwright 実装の重要注意点（配配メール）

①2画面エラー回避：編集画面への直接 URL アクセスで「不正なアクセス」が発生。必ずトップページ経由でナビゲートする。
② htmlbody 注入：ReEditGroupMailWithCustList ページに textarea が存在しないため、document.createElement('input') で hidden input として DOM に追加する。
③ 文字化け対策：HTML 内の ㎡（U+33A1）を送信前に Python 側で m² に置換する。
④ セッション管理：browser_context.storage_state() で Cookie を保存・再利用し、再ログインを最小化する。

Security Design

🔑 認証情報の移行（最優先）

スキルファイルにハードコードされている GitHub PAT は即時 Secret Manager へ移行し、ファイルから削除してください。配配メール・Lステップのログイン情報も同様に集中管理します。

🛡 最小権限のサービスアカウント

Google サービスアカウントには roles/drive.file（指定フォルダのみ）・Sheets 読み書き・Secret Manager 特定シークレット読み取り の最小権限のみ付与します。

🌐 API エンドポイント保護

外部公開エンドポイントは Bearer Token (JWT) または Cloud IAM で保護。GAS Webhook は HMAC-SHA256 署名検証 を必須とします。

🍪 Playwright セッション管理

初回ログイン後のセッション Cookie を storage_state() で保存し Secret Manager に暗号化して格納。コード・ログへのパスワード出力は厳禁です。
